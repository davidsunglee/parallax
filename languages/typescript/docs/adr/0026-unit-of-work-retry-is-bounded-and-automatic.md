# Unit-of-work retry is bounded and automatic

`px.transaction` accepts `retries` (default 10; 0 disables) and `retryOptimisticConflicts` (default false) and wraps the body in a bounded automatic retry loop: each attempt opens a fresh driver transaction and a fresh `ParallaxTransaction` (so the per-unit-of-work observed-version map is discarded and the retry re-reads fresh state). Transient database failures — Postgres SQLSTATE `40001`/`40P01`, classified through `@parallax/dialect` to the retriable `deadlock` category and surfaced across the `@parallax/db` port — are auto-retried by default; a `ParallaxOptimisticLockError` joins the retriable set only when `retryOptimisticConflicts` is set; a lock-wait timeout (`55P03`) is never retried. Conflicts stay caller-inspectable by default; the opt-in and the always-on transient retry follow the bounded-retry contract core ADR 0008 mandates.

**Considered options.** A pure caller-driven posture — surface every `ParallaxOptimisticLockError` and never auto-retry, because automatic replay can hide domain decisions, duplicate side effects, and apply stale intent over state the user has not inspected — was the original TypeScript decision. It survives as the `retryOptimisticConflicts: false` default, but was rejected as the whole model once core mandated always-on bounded retry for transient database failures.
