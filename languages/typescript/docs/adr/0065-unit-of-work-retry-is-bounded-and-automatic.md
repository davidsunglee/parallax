# Unit-of-work retry is bounded and automatic

`px.transaction` accepts `retries` (default 10; 0 disables) and `retryOptimisticConflicts` (default false) and wraps the body in a bounded automatic retry loop: each attempt opens a fresh driver transaction and a fresh `ParallaxTransaction` (so the per-unit-of-work observed-version map is discarded and the retry re-reads fresh state). Transient database failures — Postgres SQLSTATE `40001`/`40P01`, classified through `@parallax/dialect` to the retriable `deadlock` category and surfaced across the `@parallax/db` port — are auto-retried by default; a `ParallaxOptimisticLockError` joins the retriable set only when `retryOptimisticConflicts` is set; a lock-wait timeout (`55P03`) is never retried. This supersedes ADR 0048's runtime-does-not-auto-retry posture, keeping conflicts caller-inspectable by default while adding the opt-in and the always-on transient retry that core ADR 0031 mandates.
